# Getting help

Where to go depends on what you need. Pick the first one that matches.

## 🐛 Something is broken

→ Open a **bug report** on the repo where you hit it.
Use the [bug template](https://github.com/flytohub/.github/blob/main/.github/ISSUE_TEMPLATE/bug_report.md) — repro steps + version save us a round-trip.

## 💡 You want a feature

→ Open a **feature request** on the relevant repo.
[Feature template here.](https://github.com/flytohub/.github/blob/main/.github/ISSUE_TEMPLATE/feature_request.md) Describe the *problem* first, the solution second.

## ❓ How do I…?

→ [GitHub Discussions](https://github.com/flytohub/flyto-core/discussions) — Q&A category.
Faster than email, public so the answer helps the next person too.

## 📖 Reading the docs

→ [docs.flyto2.com](https://docs.flyto2.com) — full product documentation.
→ Each repo's `README.md` — quick-start + architecture.
→ [flyto-blog](https://github.com/flytohub/flyto-blog) — design decisions, deep dives.

## 📺 Show me, don't tell me

→ [YouTube @Flyto2](https://www.youtube.com/@Flyto2) — demos, walkthroughs, release tours.

## 🔒 Security issue

→ **Do not open a public issue.** See [SECURITY.md](./SECURITY.md).
📧 security@flyto2.com — acknowledged within 24 hours.

## 🤝 Code of conduct concern

→ conduct@flyto2.com — goes to a rotating group of maintainers, not a single person.

## 💼 Commercial / enterprise / press

→ hello@flyto2.com
- Pro / enterprise trials & POCs
- Partnership & integration enquiries
- Press, podcasts, conference invitations

## ⏱️ How fast do you respond?

| Channel | Target |
|---------|--------|
| Security report | 24 hours acknowledgement, 72 hours triage |
| Bug / feature issue | 1–3 business days |
| Discussions Q&A | Best-effort by maintainers + community |
| Conduct report | 72 hours |
| Commercial enquiry | 2 business days |

## What we *don't* do here

- **No private 1-on-1 free support.** OSS questions go in Discussions so everyone benefits.
- **No DMs on social.** Twitter / LinkedIn replies happen when they happen — for anything substantive, use the channels above.
- **No support for forks we didn't publish.** If you're running a fork, open issues there first.
